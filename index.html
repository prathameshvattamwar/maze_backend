Maze Game Backend Plan

Step-by-step guide to implementing the backend for a maze game using Node.js and MySQL.

Database Setup

Use the following SQL schema to create the necessary tables:

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    first_name VARCHAR(100),
    last_name VARCHAR(100),
    email VARCHAR(150) UNIQUE,
    password VARCHAR(255),
    verification_code VARCHAR(6),
    is_verified BOOLEAN DEFAULT FALSE
);

CREATE TABLE password_resets (
    id INT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(150),
    reset_code VARCHAR(6),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Workflow

Signup Flow

  1. User fills in the signup form (First Name, Last Name, Email, Password).
  2. Click "Send Verification Code":
    • Backend generates a 6-digit code.
    • Saves user data to the database.
    • Sends code to user's email.
  3. User enters verification code and clicks "Signup":
    • Code is verified against the database.
    • Password is hashed, and `is_verified` is updated.
    • User is redirected to the login page.

Login Flow

  1. User inputs email and password.
  2. Backend verifies credentials and `is_verified` status.
  3. On success, redirects user to the maze game.

Forgot Password Flow

  1. User clicks "Forgot Password?" and enters email.
  2. Backend generates a reset code and emails it.
  3. User clicks the reset link, fills in the new password, and saves:
    • Password is updated in the database.
    • Redirects user to the login page.
